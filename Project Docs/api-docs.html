Angstrom - Student Mini Project

Base URL

https://d2c6afifpk.execute-api.eu-west-2.amazonaws.com/dev

This collection provides access to the Angstrom dataset exposed through Amazon Athena via a RESTful API. Each endpoint supports paginated access to data using queryExecutionId, nextToken, and columns to enable efficient retrieval of large datasets without re-running queries.

Available Endpoints

  • GET /players: Player profiles and metadata
  • GET /fixtures: Historical match fixture information
  • GET /boxscores: Match-level player performance statistics
  • GET /three_pt_contest_historical_results: NBA 3-point contest results by player and year

Pagination

All endpoints support pagination using the following pattern:

  1. First request � no query parameters
    Returns: results, queryExecutionId, nextToken, and columns
  2. Subsequent requests � use:
    • queryExecutionId from the first response
    • nextToken from the previous response
    • columns from the first response (as a URL-encoded JSON array)

Example: Paginated Request

After retrieving the first page from /players, you'll receive a queryExecutionId, a nextToken, and a list of columns to use for subsequent pages.

Example call to get the next page of results:

GET https://d2c6afifpk.execute-api.eu-west-2.amazonaws.com/dev/players?queryExecutionId=294aa831-4f52-4992-84f2-51ac98968655&nextToken=AR8bP%2BpnYaxGrAclbv16DWynUqm9yGsEV3H1ts%2FSGZew2XOPf8F5V0Qucr3mVFOlc1REP8LBRDO6y1fZEnFys9tPquxEYgEY8Q%3D%3D&columns=%5B%22row%22%2C%22playerid%22%2C%22firstname%22%2C%22surname%22%5D

Be sure to URL-encode the nextToken and columns parameters, as shown above.
